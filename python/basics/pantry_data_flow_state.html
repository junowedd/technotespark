Pantry Data Flow & State Management

SmartShelf operates on a lightweight state model built around Pandas DataFrames. All pantry and shopping data originate from CSV storage.

Data Flow Overview

CSV → load_csvs()
    → Pandas DataFrames (userPantry, shoppingList)
    → UI Interaction
    → Data Update
    → closeout() → Save CSV

State Handling Strategy

  • Global DataFrame references for runtime state
  • Immediate in-memory updates upon user action
  • Persistence triggered during application exit

Example: Updating Quantity


userPantry.loc[
    userPantry['item_name'] == original_name,
    'quantity'
] = qty_val

This direct DataFrame mutation keeps state consistent across UI rendering cycles.

Design Characteristics

  • Simple, readable state model
  • No complex ORM or backend dependency
  • Suitable for small-to-medium interactive tools
  • Easily extendable to database-backed persistence
